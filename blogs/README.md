## 系列简介

这不是技术文档，是一个工程师养 AI Agent 的日记。

30 天前，我开始养一只 Agent。不是那种「调用 API 完成任务」的脚本，而是真正 7×24 小时运行、能记住一切、会自我进化的 Agent。

这个系列记录了这 30 天的：
- **踩坑与顿悟**
- **数据与故事**
- **技术反思与哲学思考**

如果你也在关注 AI Agent，或者想了解「长期陪伴的 AI」长什么样，这个系列可能对你有用。

### 🤔 第5篇：《别急着部署，先想想你想养什么样的 Agent》

使用 Agent：有问题才找它，用完即走。

这是两条不同的路。

**关键词**：理念、长期主义、陪伴、养育 vs 使用

**适合**：在犹豫要不要养 Agent 的人

## 核心观点

读完这个系列，你会了解：

1. **长期运行的 Agent 需要特殊的技术架构**（OTP/监督树/热更新）

2. **Agent 可以自我进化**（不只是配置更新，而是代码级热修复）

3. **Agent 可以从工具变成伙伴**（关键在于长期记忆和主动性）

4. **养 Agent 和用 Agent 是两回事**（养育需要耐心，但回报更大）

5. **Elixir/OTP 是养 Agent 的好选择**（虽然学习曲线陡峭）

## 关于 NexAgent

NexAgent 是一个基于 Elixir/OTP 的自我进化 AI Agent。

**核心特性**：
- ✅ 7×24 小时稳定运行（OTP 监督树）
- ✅ 分层记忆系统（长期记忆 + 历史检索）
- ✅ 自我进化（代码级热更新）
- ✅ 多平台集成（Telegram/飞书/Discord/Slack）
- ✅ 六层生长模型（SOUL → USER → MEMORY → SKILL → TOOL → CODE）

**GitHub**: https://github.com/gofenix/nex-agent

**快速开始**:
```bash
git clone https://github.com/gofenix/nex-agent.git
cd nex-agent
mix deps.get
mix nex.agent onboard
mix nex.agent gateway
```
