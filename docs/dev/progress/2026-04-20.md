## Session run control design

- 调查了 `openclaw` 和 `hermes-agent` 的 busy / interrupt / stop 实现：
  - Hermes 默认新消息 interrupt，`/queue` 不 interrupt，`/status` busy bypass，工具用 per-thread interrupt。
  - OpenClaw 有 run registry / AbortController / `chat.abort` / stop primitives / lane queue / embedded run handle。
- 结论调整为更符合 IM 聊天直觉的设计：
  - busy 普通消息默认 follow-up，不打断 owner run
  - `/stop` 是 deterministic hard control
  - `/queue` 显式排队
  - `/btw` 显式旁路提问
  - `/status` 直接读 run snapshot
- 已撤回临时实现改动，没有保留半成品代码。

## Phase 8 implementation

- 已完成 `Nex.Agent.RunControl`：
  - owner run start / finish / fail / cancel
  - run id / cancel ref
  - latest tool tail / assistant partial / queued count snapshot
- `InboundWorker` 已切到 owner run 语义：
  - `active_tasks` 不再只存 pid
  - async result / stream state 按 run id 隔离
  - stale late result 会被丢弃
- busy session 普通消息不再默认排队进 owner run：
  - 直接走 follow-up reply
  - `/status` deterministic 读 run snapshot
  - `/btw` deterministic side answer
  - `/queue` 明确排队，并在 idle 时立即拉起下一 owner run
- `/stop` 现在是 deterministic hard control：
  - 先 cancel owner run
  - 再取消 owner task / active tool tasks / subagent tasks
  - stop 后旧 run late result 会被 run id 丢弃
- `Runner` / `Tool.Registry` / long tool 取消 contract 已接入：
  - runner opts / tool ctx 带 `run_id` / `cancel_ref`
  - LLM retry sleep / iteration / tool execution 前会检查 cancel
  - `Tool.Registry` 记录 active tool task，并支持 `cancel_run/1`
  - `bash` 改成可取消外部 shell process
  - `web_fetch` / `web_search` 至少支持 cancel 前后早退
- `Subagent` 现在带 `owner_run_id`，`/stop` 可按 owner run 取消相关 subagent。
