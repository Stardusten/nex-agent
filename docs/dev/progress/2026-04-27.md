# 2026-04-27

## File Access Allowed Roots

- 新增 `tools.file_access.allowed_roots` runtime config contract，用于单独白名单授权非 workspace 目录。
- `workspace` 保持单一 agent home，不支持数组，也不承载多项目文件访问语义。
- `Nex.Agent.Security` 删除隐式 path validation 入口，只保留显式上下文 API：
  - `allowed_roots(ctx)`
  - `validate_path(path, ctx)`
  - `validate_write_path(path, ctx)`
- `read`、`find`、`apply_patch` 统一从 tool ctx 传入 runtime config / workspace，再走 Security 校验。
- root 边界检查修正为路径边界匹配，避免 `/repo` 误放行 `/repo-other`。
- `NEX_ALLOWED_ROOTS` 仍作为进程级完整 override。

## Model Context Window And Native Compaction

- 模型 runtime 现在解析 `context_window`、`auto_compact_token_limit`、`context_strategy`，这些字段不再混入 provider request options。
- 新增 `Nex.Agent.ContextWindow` 作为唯一 context projection 边界：先按模型窗口选择 history，再生成 provider-native compaction 选项。
- `Runner` 的初始请求和 tool-loop 递归都复用该 projection；OpenAI Codex 原生 compaction item 会进入 response metadata、session metadata 和下一次请求 input。
- OpenAI Codex Responses payload policy 支持注入 `context_management` 和 opaque `compaction` input item。
