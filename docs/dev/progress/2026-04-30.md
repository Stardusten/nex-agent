## Phase 21 Stage 2 Approval State And Command Lane

产出：

- 新增 approval 主链：
  - `Nex.Agent.Sandbox.Approval`
  - `Nex.Agent.Sandbox.Approval.Request`
  - `Nex.Agent.Sandbox.Approval.Grant`
  - `Nex.Agent.Sandbox.Permission`
- `Approval` 支持：
  - pending request FIFO
  - allow once
  - allow all current-session pending once
  - session grant
  - similar session grant via safe family grant option
  - always grant persisted to `<workspace>/permissions/grants.json`
  - deny once / deny all
  - reset session cancels pending and clears session grants
  - stop cancels pending without clearing session grants
- `Workspace.ensure!/1` now creates `permissions/`。
- `InfrastructureSupervisor` starts `Nex.Agent.Sandbox.Approval`。
- `/approve` / `/deny` are declared in `priv/plugins/builtin/command.core/nex.plugin.json` and bound through the bounded command handler table。
- `InboundWorker` handles `/approve`, `/approve all`, `/approve session`, `/approve similar`, `/approve always`, `/deny`, `/deny all` deterministically and bypasses busy LLM routing。

边界：

- Stage 2 仍不改变 tool filesystem 或 bash execution 行为。
- Approval request prompt 通过 Bus/outbound topic 发送，不走 `message` tool。
- Workspace/project command plugins 仍不能绑定 executable handler；`approve`/`deny` 只来自 builtin bounded handler。

## Phase 21 Stage 3 Direct File Permission Cutover

产出：

- 新增 `Nex.Agent.Sandbox.FileSystem` 作为 user/model-controlled path 的统一 filesystem authority。
- `Nex.Agent.Sandbox.Security.authorize_path/3` 现在基于 `Runtime.Snapshot.sandbox` / `Config.sandbox_runtime/2` 做 permission decision：
  - canonical path info
  - symlink-aware allowed-root checks
  - missing write target nearest existing ancestor checks
  - protected paths hard deny
  - interactive approval request / once / session grant resume
- 已迁移 direct file-like tools：
  - `read`
  - `find` scope path validation
  - `apply_patch`
  - `message` attachment/local image path
  - `reflect source path`
  - `user_update`
  - `soul_update`
  - `memory_write`
- Feishu outbound attachment upload now consumes authorized attachment metadata or re-authorizes through `Sandbox.FileSystem.stream_file/2` instead of raw `File.stream!`。
- `NEX_ALLOWED_ROOTS` remains a process-level override during migration, but the main runtime truth source is now sandbox policy。
- Added static inventory coverage for migrated direct file tools so reviewed paths cannot silently reintroduce direct `File.*` access。

边界：

- `find` still starts `rg` through `System.cmd`; Stage 6 will move that child process through `Sandbox.Exec.run/2`。
- Internal fixed workspace helpers such as general memory refresh/rebuild still use direct `File.*` as reviewed CODE-layer/internal IO; model-visible `memory_write` now passes through the permission layer。
- Stage 3 does not implement OS process sandboxing for `bash` yet。

## Phase 21 Stages 4-7 Exec, Bash Approval, Process Migration

产出：

- 新增 process sandbox 主链：
  - `Nex.Agent.Sandbox.Backend`
  - `Nex.Agent.Sandbox.Exec`
  - `Nex.Agent.Sandbox.Process`
  - `Nex.Agent.Sandbox.Backends.Noop`
  - `Nex.Agent.Sandbox.Backends.Seatbelt`
- `Sandbox.Exec.run/2` owns completed command execution：
  - backend selection / fail-closed behavior
  - env allowlist filtering and explicit unsetting of non-allowlisted inherited env
  - stdin tempfile piping
  - timeout / cancellation polling
  - bounded UTF-8-safe output sanitization
  - ControlPlane `sandbox.exec.*` lifecycle observations
- `Sandbox.Exec.open/2` / `write/2` / `close/1` now provide the long-running stdio process contract required by MCP。
- macOS Seatbelt backend uses fixed `/usr/bin/sandbox-exec` and generates a deny-default profile with:
  - unrestricted read minus protected/denied path exclusions
  - write allow only for sandbox policy write roots
  - network disabled unless policy enables it
- `bash` no longer opens `sh` directly; it now:
  - classifies commands with `Sandbox.CommandClassifier`
  - requests approval for approval-required commands
  - supports exact/session/similar command grants
  - executes through `Sandbox.Exec.run/2`
- Migrated remaining child-process call sites:
  - `find` runs `rg` through `Sandbox.Exec.run/2`
  - external executor runs through `Sandbox.Exec.run/2`
  - stdio MCP servers start through `Sandbox.Exec.open/2`
  - self-update git/test subprocesses route through `Sandbox.Exec` with an internal no-sandbox policy
- Removed git subprocess probes from project/context detection by replacing `git rev-parse` with parent `.git` discovery.
- Approval lifecycle now records `sandbox.approval.*` ControlPlane observations without logging raw command/path contents.

Reviewed remaining subprocess exemptions:

- `Nex.Agent.Sandbox.Exec` is the only module that owns raw `Port.open/2`.
- `Nex.Agent.Observe.Admin` uses `kill -0` only as an internal pid liveness probe.
- `Mix.Tasks.Nex.Agent` uses `kill` commands only for local CLI gateway process management, outside model/tool execution.

## Phase 21 Discord Approval Components

产出：

- 新增 `Nex.Agent.Interface.Outbound.Approval` 作为 channel-agnostic approval control payload。
- `Sandbox.Approval` 发布 pending request 时仍走同一条 Bus/outbound lane，但 metadata 现在带规范化 approval request/actions。
- Discord channel 识别 approval payload 并发送 Component V2 消息：
  - `Approve once` -> `/approve`
  - `Allow command` -> `/approve session`
  - `Allow similar` -> `/approve similar` when the request has a safe similar grant option
  - `Always allow` -> `/approve always`
  - `Decline` -> `/deny`
- Discord `MESSAGE_COMPONENT` interaction click 会 ack component update，并转成同一条 deterministic slash command inbound envelope；approval 状态机没有新增平行状态。
- Discord 原生 approval prompt 隐藏 slash command fallback 用法；按钮点击后用 interaction type 7 更新原组件消息，保留 approval request 文本并追加状态行，同时移除按钮。
- `bash` tool 的成功结果现在会告诉 LLM 命令是用户刚批准后执行，或命中了已有 approval grant，避免模型误判为“未触发审批”。
- 高风险 shell 形态从历史硬黑名单迁移到 exact approval：
  - command/process substitution、shell escape、interpreter one-liner、`base64 | sh` encoded shell pipeline 都会携带 risk hint。
  - 这些命令不吃 sandbox default allow；必须有显式 approval 或已有 exact grant。
  - truly destructive/system commands such as `sudo`、磁盘操作、删系统目录、fork bomb 仍 hard deny。
