# 2026-04-26

## Session Model Command

- 新增 `/model [name|number|reset]` 作为用户控制命令，不进入 LLM/tool 主链。
- 当前会话的模型 override 存在 session metadata，解析时仍只接受 `config.model.models` 中的 `model_key`，不允许裸 provider/model id 绕过配置真相源。
- InboundWorker 创建会话 agent 时按 `session override -> default_model -> first available` 解析模型；切换模型会清掉该 session 的 cached agent，使下一轮 owner/follow-up run 使用新模型。正在运行的 owner run 不被打断。
- `/model` 列表使用 IM 友好的 Markdown：当前项用 blockquote + bold，模型支持稳定数字 id，因此 `/model 1` 可直接切换。
- `/model reset` 和 `/new` 都清除 session model override；`/new` 同时清空当前 session 消息和 consolidation state。

## Compact Status View

- `/status` 改为短 Markdown 状态板，默认包含：
  - owner run 状态 / phase / elapsed / queue
  - 当前会话模型与 override 来源
  - 估算 context 使用量；若模型配置含 `context_window` / `context_tokens` / `max_context_tokens` / `context_limit` 则显示剩余，否则显示 limit unknown
  - enabled channel instance 的 connected/disconnected 状态
  - 前 6 个 model id，超出时提示 `/model` 查看完整列表
  - 原有 ControlPlane warning/error evidence 摘要

## Discord Table Rendering

- Discord channel 新增 `show_table_as` 配置，允许 `raw` / `ascii` / `embed`，默认 `ascii`。
- `ascii` 将 Markdown table 渲染成 fenced monospace 字符表，避免依赖 Discord 客户端 embed preview。
- `raw` 保留原始 Markdown table；`embed` 使用 Discord embed fields，并保留可见 fallback 文本。
- runtime channel context 会把当前表格模式写进 Discord 格式引导；Discord outbound observation 增加 `table_mode`。

## Unified Runtime Hooks

- `session_prompt` / `pinned_prompt` 被统一 hook registry 取代；session 语义现在通过 `hooks/hooks.json` 里的 `prompt.build.before` hook 表达。
- 新增 `hook` local tool 管理 `hooks/hooks.json`，支持 `list` / `show` / `add_file` / `add_text` / `enable` / `disable` / `remove` / `test`。
- Runtime snapshot 加载 hooks，Watcher 监听 `hooks/hooks.json` 并热重载；Runner 在 build messages 前匹配 hook，ContextBuilder 将 hook fragments 注入 system prompt。
- v1 hook 只支持 `file` / `text` advice，按 `session` / `channel` / `chat_id` / `workspace` 精确 pointcut 匹配。
