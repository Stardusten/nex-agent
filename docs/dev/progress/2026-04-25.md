# 2026-04-25

- 删除 `web_search` / `image_generation` 上一轮引入的 native-provider lane / capability resolver / builtin tool 语义，收回到单一主链：模型只见 local tool，运行时只执行 local tool。
- `Tool.Registry`、`ReqLLM`、`Runner` 不再为这两类能力保留 provider-native 或 builtin-tool 分支；删除 `Capability`、`CapabilityResolver` 和相关 adapter 文件与测试。
- `Config` 把两类工具配置改成 provider 风格：
  - `web_search_provider_config/1`
  - `image_generation_provider_config/1`
  - shape 为 `provider + providers`
- `web_search` 和 `image_generation` 统一改为“tool 内部按配置选择 backend”：
  - `web_search`: `duckduckgo` / `codex`
  - `image_generation`: `codex`，并给未来 `nanobanana` 留 provider 位但不实现 backend
- focused tests 重新对齐到新 contract：验证 local function tool contract、provider 风格配置入口、以及 Codex backend 下仍然只向模型暴露普通 function tool。
- 沉淀架构结论到 findings：
  - 这类能力的冻结边界是 `local tool façade + backend implementation`
  - 不要再把 backend 选择做成 provider-native lane
  - 后续 review 要主动拦截第二工具链路语义回流

## 新配置结构一步切换

- `Config` 真相源切到新外部 shape：顶层只保留 `channel`、`gateway`、`provider`、`model`、`tools`、`workspace`、`max_iterations`；旧顶层 `provider`/`model` 字符串、`providers`、`defaults`、顶层渠道字段不做兼容。
- 新增 model role runtime 解析：`default_model_runtime/1`、`cheap_model_runtime/1`、`advisor_model_runtime/1`，模型 entry 的 provider/provider type/base_url/api_key/options 都从 `provider.providers + model.models` 收口。
- Gateway / Runtime / Runner / InboundWorker / Admin / Mix task 改为通过新 accessor 取默认模型 runtime，不再从业务层读旧 JSON shape。
- channel 改为 instance id 架构：
  - `Config.channel_instances/1` / `channel_instance/2`
  - Gateway 一个 enabled channel entry 起一个 child，child id 为 `{module, instance_id}`
  - Feishu / Discord process 通过 `Nex.Agent.Interface.Channel.Registry` 注册，不再是 module singleton
  - inbound envelope 的 `channel` 是 instance id，`metadata["channel_type"]` 存真实渠道类型
  - outbound topic 改为 `{:channel_outbound, instance_id}`
- `message` tool 的 `channel` 参数改为 channel instance id；默认目标仍是当前 inbound instance。
- `tools.web_search` / `tools.image_generation` 最终收口为 provider-style backend contract，不保留 `strategy` vocabulary；历史 flat contract 已删除。

## Subagent profile MVP

- 将现有 generic background subagent 推进到 profile-aware：
  - 新增 `Nex.Agent.Capability.Subagent.Profile` / `Profiles`
  - 内置 `general`、`code_reviewer`、`researcher`
  - 支持 config `subagents.profiles` 与 workspace `subagents/*.md`
- Runtime snapshot 新增 `subagents.profiles/definitions/hash`，`spawn_task` tool definition 可根据当前 runtime profiles 暴露 `profile` enum 与 profile descriptions。
- `spawn_task` model-visible 参数收窄为 `profile`、`task`、`label`、`context`；context mode、return mode、model role/key 归 profile contract 所有。
- `Subagent` 按 profile 组装 prompt、模型 runtime、provider options、tool surface、tool allowlist、父会话 recent context 与结果投递方式；子会话默认跳过 memory consolidation，并持久化 child session。
- 修正 owner run 控制链路：InboundWorker/Nex.Agent.prompt 现在把 `run_id` 与 `cancel_ref` 传给 Runner，subagent owner cancellation 和 tool cancellation 能看到同一 run/cancel 引用。
- Review simplification:
  - profile 不再允许 `base` tool surface；Runner snapshot fallback 的 `:base` 不再回退到 all tools。
  - profile string config 不再动态创建 provider option / model role / tools filter atoms。
  - `return_mode` 收敛为 `inbound | silent`，silent 不发布 visible inbound 或 subagent bus event。
  - subagent runtime hash 基于完整 executable profile，而不是只基于 model-visible definitions。

## Local advisor tool

- 新增 phase16 执行计划：advisor 采用本地 tool，不走 provider-native lane，不复用 `spawn_task` 的异步投递语义。
- 新增 `ask_advisor` owner tool：
  - 同步返回建议给当前 runner，不自动发送用户可见消息。
  - 支持 `context_mode=full|recent|none`，并允许 caller 追加手工 `context`。
  - 默认使用本地 advisor profile（`model_role: :advisor`、无工具、`max_iterations: 4`），运行时自定义 `advisor` profile 可覆盖 prompt/model/provider options。
  - `model_key` 可单次覆盖到 config model runtime。
- Runner tool ctx 传递测试/本地 LLM 注入函数，保证工具内部启动的 advisor run 仍走同一测试入口。
- prompt guidance 新增 `ask_advisor` 使用时机：计划复核、卡住诊断、风险判断；advisor output 仅作为 run 内部指导。
