# Designs Index

`docs/dev/designs/` is for product and architecture design work before it becomes an executable phase plan.

Use this directory for:

- competing options and tradeoffs
- rough architecture sketches
- product behavior proposals
- open questions that are not ready to freeze
- design notes that may later produce findings or task plans

Do not use this directory for:

- daily progress records
- final architecture decisions that should guide reviews
- phase execution steps with file paths and test commands

When a design settles, promote the stable decision into `docs/dev/findings/`. When it becomes implementation work, create or update a concise `docs/dev/task-plan/` file that contains only execution stages, frozen contracts, acceptance checks, and validation commands.
